# LocalTwin v0.1 구현 범위 고정 명세

## 1. 프로젝트 정의

> LocalTwin은 공공데이터 기반 상권 분석 대시보드를 주기능으로 구현하고, 보조/추가기능으로 한 가게 앞 또는 한 거리 10~20m 구간의 혼잡도와 3D 기반 탐색을 제공하는 웹 기반 상권 디지털 트윈 프로토타입이다.

핵심은 도시 전체를 3D로 복원하는 것이 아니다.

```text
공공데이터 기반 상권 분석
+ 보조/추가기능으로 제공하는 한 장면 3D 기반 탐색
+ 시간대별 혼잡도 관찰 데이터
= 소상공인/예비창업자 입지 판단 보조 도구
```

## 2. 결정 흐름

초기 선택지는 다음 아이디어였다.

```text
골목상권 3D/AR 탐색 앱:
실제 골목/가게 앞 공간을 3D로 재구성하고,
그 위에 가게 정보, 상권 데이터, 길찾기, 입지 정보를 얹는 서비스
```

단, v0.1에서는 AR과 도시 규모 복원을 제외한다. 주기능은 상권 분석이고, 3D 기반 탐색은 혼잡도와 현장감을 보여주는 보조/추가기능으로 둔다.

초기 촬영 전략의 출발점은 다음 판단이다.

```text
3D 공간 복원용 촬영은 한 번만 한다.
10시 / 13시 / 15시 / 18시 촬영은 3D 학습용이 아니라 상권 변화와 혼잡도 분석용으로 쓴다.
```

이후 프로젝트 범위를 다음처럼 재정의한다.

```text
주기능:
공공데이터 기반 소상공인 상권 분석 대시보드

보조/추가기능:
한 가게 앞 또는 한 거리 10~20m를 직접 촬영해 3D 기반 탐색과 혼잡도 오버레이 제공
```

이렇게 나누는 이유는 상권 분석이라는 서비스의 중심을 지키면서도 3D 공간정보 기술을 보여줄 수 있기 때문이다.

## 3. 역할 분리

### A. 주기능: 공공데이터 기반 상권 분석

대상 범위는 특정 상권 1곳으로 제한한다. v0.1 명세에서는 아직 서울로 고정하지 않는다.

예시:

```text
대학가 상권
역세권 상권
골목상권
카페 밀집 상권
```

제공 기능:

```text
상권 지도
업종별 점포 수
동일 업종 경쟁 강도
개업/폐업 추세
생활인구 시간대 변화
입지 점수
AI 상권 해석 리포트
```

이 기능이 사용자가 실제로 쓰는 핵심 서비스다.

### B. 보조/추가기능: 혼잡도 3D 기반 탐색

직접 촬영/3D 복원은 프로젝트 전체의 중심이 아니라 상권 분석을 보강하는 보조/추가기능으로 제한한다.

대상 범위:

```text
대전 유성구 관평동 한 장소
점포 전면 또는 보도 10~20m
```

제공 기능:

```text
스마트폰으로 촬영한 공간을 3D로 복원
웹에서 3D로 탐색
가게 위치에 마커 표시
마커 클릭 시 공공데이터 기반 상권 정보 카드 표시
시간대 버튼 클릭 시 혼잡도/관찰 데이터 표시
```

3D는 모든 상권을 복원하는 기능이 아니라, 선택한 상권의 일부 현장을 탐색하고 혼잡도 데이터를 공간 위에서 이해하게 돕는 보조 기능이다.

## 5. 촬영 범위

처음에는 한 가게 앞 10~20m를 추천 범위로 고정한다.

포함 대상:

```text
작은 카페 1곳
가게 입구
간판
가게 앞 보도
양옆 점포 일부
골목 입구 일부
```

촬영 범위를 넓히지 않는 이유:

```text
3D 복원 실패 가능성 증가
사람/차량/반사물 증가
촬영 데이터 관리 어려움
좌표 정렬 어려움
웹 로딩 무거워짐
공개 포트폴리오 개인정보 이슈 증가
```

## 6. 촬영 계획

### A. 3D 복원용 촬영

목적:

```text
깨끗한 기본 3D 공간 생성
```

방식:

```text
사람 적은 시간 1회
가게 앞 10~20m
3~5분 촬영
천천히 이동
가게 입구, 간판, 벽면, 보도, 주변 고정 구조물이 겹쳐 보이게 촬영
```

권장 시간:

```text
08:00~09:00
또는
21:00 이후
```

### B. 시간대 분석용 촬영

목적:

```text
하루 전체를 계속 찍지 않고, 붐비는 대표 시간대만 골라 유동량/혼잡도/체류 흐름을 비교
```

촬영 시간:

```text
10:00 오전 유동
13:00 점심 이후
15:00 오후 카페/체류
18:00 퇴근/저녁
```

방식:

```text
각 시간대 1~2분
같은 위치
같은 각도
가능하면 같은 촬영 지점 A/B/C 반복
```

분석 항목:

```text
사람 수
혼잡도 등급
가게 앞 체류 여부
배달/차량 흐름
시간대별 상권 성격
```

v0.1에서는 수동 관찰값 입력을 허용한다. YOLO 같은 객체 탐지는 v0.2 이후 선택 확장으로 둔다.

## 7. 공공데이터 후보

v0.1에서 사용할 데이터 후보는 다음으로 둔다. 실제 API 필드, 제공 방식, 이용 조건은 구현 직전에 다시 확인한다.

| 데이터                             | 사용 목적                                 |
| ---------------------------------- | ----------------------------------------- |
| 소상공인시장진흥공단 상가업소정보  | 주변 점포, 업종 분포, 동일 업종 경쟁 강도 |
| 지방행정 인허가정보                | 개업/폐업 흐름, 영업상태, 업종 변화       |
| 생활인구/유동인구 데이터           | 시간대별 수요, 평일/주말 유동 특성        |
| V-World 공간 데이터                | 용도지역, 필지/공간 맥락                  |
| 네이버 Maps API 또는 대체 지도 API | 2D 지도, 마커, 반경 표시, 주소/좌표 변환  |

데이터 소스의 최신 제공 조건은 아직 이 문서에서 검증하지 않는다.

## 9. 입지 점수 v0.1

처음부터 머신러닝 모델로 시작하지 않는다. v0.1은 설명 가능한 규칙 기반 점수로 시작한다.

```text
입지 점수 = 수요 점수 + 경쟁 점수 + 변화 점수 + 공간 점수 + 시간대 점수
```

| 항목        | 데이터                          |
| ----------- | ------------------------------- |
| 수요 점수   | 생활인구, 직접 촬영 유동량      |
| 경쟁 점수   | 상가업소정보 기반 동일 업종 수  |
| 변화 점수   | 인허가정보 기반 개업/폐업 흐름  |
| 공간 점수   | 용도지역/필지 정보              |
| 시간대 점수 | 10시/13시/15시/18시 관찰 데이터 |

예시:

```text
수요 점수: 25/30
경쟁 점수: 12/25
변화 점수: 14/20
공간 점수: 10/10
시간대 점수: 12/15
총점: 73/100
```

## 11. 개인정보 및 공개 범위

공개 포트폴리오를 전제로 다음을 지킨다.

```text
원본 영상은 공개하지 않는다.
얼굴과 차량 번호판은 블러 처리한다.
사람이 개인 식별되지 않을 정도로 멀리서 촬영한다.
대시보드에는 원본 영상보다 관찰 결과와 집계값을 중심으로 표시한다.
```

## 12. v0.1 완료 기준

v0.1은 다음 조건을 만족하면 완료로 본다. 아래 항목은 최종 완료 기준이며 현재 구현 완료를 뜻하지 않는다.

```text
특정 상권 1곳을 선정했다.
카페 또는 음식점 1개 업종 기준 분석이 가능하다.
지도에서 반경 내 점포와 동일 업종 경쟁 강도를 볼 수 있다.
개업/폐업 흐름과 생활인구 시간대 그래프를 볼 수 있다.
한 가게 앞 또는 거리 10~20m의 3D 장면을 웹에서 탐색할 수 있다.
3D 장면에 가게 마커가 있고, 클릭 시 상권 정보 카드가 열린다.
10시 / 13시 / 15시 / 18시 관찰 데이터가 화면에 표시된다.
규칙 기반 입지 점수와 AI 해석 리포트를 생성할 수 있다.
```

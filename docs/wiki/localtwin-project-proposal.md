# LocalTwin 프로젝트 기획서

문서 역할: 발표와 소개를 위한 요약

상세 제품 요구사항은 [제품 기획서](./localtwin-product-plan.md), 개발 순서와 상태는 [4주 개발 백로그](../development/tasks.md)를 기준으로 한다.

## 1. 한 줄 정의

LocalTwin은 공공데이터를 기반으로 후보 상권의 점포 밀도, 경쟁 강도, 개폐업 흐름, 시간대별 수요를 한 화면에서 분석하고, 보조적으로 제한된 3D 공간 탐색을 연결하는 상권 분석 서비스다.

프로토타입:
[핵심 상권 분석 화면 프로토타입](../prototypes/core-market-analysis-prototype.html)

## 2. 해결하려는 문제

예비창업자와 소상공인은 후보 입지를 판단할 때 다음 정보를 따로 확인해야 한다.

| 문제 | 현재 사용자가 겪는 일 | LocalTwin의 해결 방향 |
| --- | --- | --- |
| 점포 분포 파악 | 지도에서 직접 점포를 하나씩 확인 | 지도 위 업종별 점포 마커와 반경 집계 제공 |
| 경쟁 강도 판단 | 같은 업종이 많은지 감으로 판단 | 100m/300m/500m 반경별 동일 업종 수와 밀도 제공 |
| 상권 변화 이해 | 개업/폐업 데이터를 따로 검색 | 최근 개업, 폐업, 순증감 흐름 요약 |
| 시간대별 수요 판단 | 현장 방문이나 후기 의존 | 생활인구/관찰값 기반 시간대 그래프 제공 |
| 판단 근거 정리 | 자료를 모아 직접 해석 | 입지 점수와 템플릿 리포트 제공 |

## 3. 대상 사용자

### 3.1 1차 사용자

```text
예비창업자
소상공인
상권 후보지를 비교하는 개인 또는 팀
```

### 3.2 사용자가 원하는 답

```text
이 위치에 같은 업종이 얼마나 많은가?
수요가 특정 시간대에 몰리는가?
최근 개업과 폐업 흐름은 어떤가?
이 상권은 카페/음식점 후보지로 위험한가, 괜찮은가?
판단 근거를 다른 사람에게 설명할 수 있는가?
```

## 4. 사용자 관점 동작 시나리오

### 시나리오 A. 카페 창업 후보지 검토

1. 사용자가 상권 검색에서 후보 상권을 선택한다.
2. 업종 필터를 `카페`로 선택한다.
3. 반경을 `300m`로 설정한다.
4. 지도에서 카페 점포 분포와 경쟁 밀도를 확인한다.
5. 개업/폐업 흐름과 시간대별 수요 그래프를 본다.
6. 입지 점수와 리포트를 읽고 위험 요인을 확인한다.
7. 필요하면 3D 보조 탐색으로 실제 거리 분위기를 확인한다.

### 시나리오 B. 음식점 후보지 비교

1. 사용자가 같은 상권에서 업종을 `음식점`으로 바꾼다.
2. 반경을 `100m`, `500m`로 바꾸며 경쟁 범위 변화를 비교한다.
3. 시간대별 수요가 점심형인지 저녁형인지 확인한다.
4. 리포트의 추천 전략을 읽고 후보지 판단 메모를 남긴다.

## 5. 핵심 기능

### 5.1 상권 선택

| 항목 | 스펙 |
| --- | --- |
| 입력 | 상권명, 행정동, 중심 좌표 |
| 출력 | 선택 상권의 지도 중심, 분석 반경 기본값 |
| 기본값 | v0.1에서는 사전 정의된 후보 상권 목록 |
| 예외 | 데이터가 없는 상권은 분석 불가 상태로 표시 |

### 5.2 업종 필터

| 항목 | 스펙 |
| --- | --- |
| 입력 | 카페, 음식점 |
| 출력 | 지도 마커, 경쟁 강도, 리포트 지표 재계산 |
| 기준 | canonical category 기준 |
| 주의 | 원본 업종명과 내부 업종 분류를 UI에서 혼동하지 않음 |

### 5.3 반경 경쟁 분석

| 항목 | 스펙 |
| --- | --- |
| 입력 | 100m, 300m, 500m |
| 계산 | Haversine 거리 기반 반경 내 동일 업종 점포 수 |
| 출력 | 경쟁 점수, 동일 업종 수, 상위 밀집 구간 |
| 사용자 가치 | 감이 아니라 수치로 경쟁 압박을 판단 |

### 5.4 개업/폐업 흐름

| 항목 | 스펙 |
| --- | --- |
| 입력 | 일반음식점/휴게음식점 인허가 데이터 |
| 출력 | 월별 개업, 폐업, 순증감 |
| 해석 | 최근 순증가면 진입 활발, 폐업 증가면 리스크로 표시 |

### 5.5 시간대별 수요

| 항목 | 스펙 |
| --- | --- |
| 입력 | 생활인구 또는 수동 관찰값 |
| 기준 시간 | 10시, 13시, 15시, 18시 |
| 출력 | 시간대별 수요 그래프, 수요 유형 라벨 |
| 주의 | 수동 관찰값은 공공데이터와 명확히 구분 |

### 5.6 입지 점수

| 항목 | 스펙 |
| --- | --- |
| 구성 | 수요, 경쟁, 변화, 시간대 적합도 |
| 출력 | 0-100점, 항목별 근거 |
| v0.1 방식 | 규칙 기반 점수 |
| 사용자 가치 | 복잡한 데이터를 하나의 판단 카드로 요약 |

### 5.7 상권 해석 리포트

| 항목 | 스펙 |
| --- | --- |
| 입력 | 요약 지표, 점수, 시간대 특성 |
| 출력 | 위험 요인, 기회 요인, 추천 전략 |
| v0.1 방식 | template report 우선, 이후 LLM adapter 연결 |

## 6. 화면 구조

```text
┌───────────────────────────────────────────────────────────────┐
│ Top Bar: 상권 선택 / 업종 필터 / 반경 선택 / 리포트 보기        │
├───────────────────────────────┬───────────────────────────────┤
│ Map Panel                     │ Summary Panel                 │
│ - 점포 마커                   │ - 입지 점수                   │
│ - 분석 반경 원                │ - 동일 업종 수                │
│ - 선택 점포 정보              │ - 개업/폐업 흐름              │
├───────────────────────────────┼───────────────────────────────┤
│ Time Demand Chart             │ Interpretation Report         │
│ - 10/13/15/18시 수요          │ - 위험 요인                   │
│ - 시간대 유형                 │ - 추천 전략                   │
└───────────────────────────────┴───────────────────────────────┘
```

## 7. 화면 단위 동작

| 화면 단위 | 사용자 행동 | 시스템 반응 |
| --- | --- | --- |
| 상권 선택 | 후보 상권 선택 | 지도 중심, 점포, 요약 지표 재조회 |
| 업종 필터 | 카페/음식점 전환 | 마커 색상, 경쟁 수치, 리포트 재계산 |
| 반경 선택 | 100/300/500m 전환 | 분석 반경 원과 동일 업종 수 변경 |
| 점포 마커 | 마커 클릭 | 점포명, 업종, 거리, 경쟁 카드 표시 |
| 시간대 그래프 | 시간대 선택 | 해당 시간대 수요와 리포트 문구 강조 |
| 리포트 카드 | 리포트 보기 | 핵심 리스크와 추천 전략 표시 |

## 8. 보조 기능의 위치

3D 기반 탐색은 핵심 판단 이후 현장감을 보조하는 기능이다.

```text
상권 분석 완료
-> 후보 위치가 의미 있어 보임
-> 3D 보조 탐색에서 거리 분위기와 혼잡도 확인
```

v0.1에서는 3D가 분석의 필수 입력이 아니라, 한 장면을 보여주는 보조 데모다.

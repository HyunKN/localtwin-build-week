# LocalTwin 제품 기획서

문서 역할: 상세 제품 기획의 원본

이 문서는 문제 정의, 사용자 시나리오와 제품 기능을 관리한다. 개발 순서와 상태는 [4주 개발 백로그](../development/tasks.md), 기술 연결은 [시스템 아키텍처](../development/architecture.md)를 기준으로 한다.

LocalTwin의 주기능은 공공데이터 기반 상권 분석이다. 혼잡도 3D 기반 탐색은 상권 분석을 보조하는 추가 기능으로 둔다.

## 1. 문제 정의

### 1.1 사용자가 겪는 문제

예비창업자와 소상공인은 입지를 판단할 때 여러 정보를 따로 확인해야 한다.

```text
주변에 같은 업종이 얼마나 많은지
최근 개업과 폐업이 늘고 있는지
시간대별 유동이 어떤지
해당 위치가 주거형/상업형/역세권형 상권인지
실제 거리의 분위기와 혼잡도가 어떤지
```

현재는 이런 정보가 흩어져 있다.

```text
공공데이터는 표와 파일 중심이라 해석이 어렵다.
지도에서는 점포 위치는 보이지만 경쟁 강도와 변화 흐름을 바로 알기 어렵다.
현장 방문은 주관적이고 특정 시간대에 치우치기 쉽다.
촬영 영상은 개인정보와 공개 위험 때문에 그대로 서비스에 쓰기 어렵다.
```

### 1.2 LocalTwin이 푸는 문제

LocalTwin은 특정 상권 1곳을 기준으로 공공데이터와 시간대별 관찰값을 연결해, 사용자가 입지 판단에 필요한 핵심 정보를 한 화면에서 이해하도록 돕는다.

```text
상권 데이터 수집
→ 업종/반경/시간대 기준 분석
→ 입지 점수와 해석 리포트 생성
→ 필요 시 3D 기반 탐색으로 현장감 보조
```

### 1.3 v0.1 문제 범위

v0.1은 전국 서비스나 자동 창업 성공 예측을 목표로 하지 않는다.

```text
대상: 특정 상권 1곳
업종: 카페 또는 음식점 1개 업종부터
분석: 반경 내 경쟁 강도, 개폐업 흐름, 시간대별 유동 특성, 입지 점수
보조: 한 가게 앞 또는 거리 10~20m의 사람 눈높이 3DGS 현장 상세보기
```

## 2. 사용자 시나리오

### 2.1 주요 사용자

#### 예비창업자

창업 후보지를 고르기 전에 주변 경쟁점, 시간대별 수요, 개폐업 흐름을 보고 싶어 한다.

#### 기존 소상공인

현재 가게 주변의 경쟁 변화와 시간대별 유동 특성을 파악해 운영 전략을 조정하고 싶어 한다.

#### 프로젝트 평가자/포트폴리오 독자

공공데이터 분석, 지도 UI, 입지 점수, 3D 공간정보 기술이 하나의 문제 해결 흐름으로 연결되는지 확인하고 싶어 한다.

### 2.2 기본 사용 흐름

```text
1. 사용자가 분석할 상권을 선택한다.
2. 대상 업종을 선택한다.
3. 분석 반경을 100m / 300m / 500m 중 선택한다.
4. 지도에서 주변 점포와 동일 업종 경쟁점을 확인한다.
5. 상권 요약 카드에서 점포 수, 경쟁 강도, 개폐업 흐름을 확인한다.
6. 시간대별 유동 그래프 또는 관찰값을 확인한다.
7. 입지 점수와 AI/템플릿 해석 리포트를 읽는다.
8. 필요하면 3D 기반 탐색 화면에서 실제 거리 구간과 혼잡도 마커를 확인한다.
```

### 2.3 예비창업자 시나리오

```text
사용자: 카페 창업을 준비하는 예비창업자
목표: 후보 입지가 카페 상권으로 적합한지 판단
```

흐름:

```text
1. 후보 상권을 선택한다.
2. 업종을 카페로 설정한다.
3. 반경 300m를 선택한다.
4. 동일 업종 점포 수와 디저트/음료 계열 점포 분포를 확인한다.
5. 최근 개업/폐업 흐름을 본다.
6. 10시 / 13시 / 15시 / 18시 유동 특성을 비교한다.
7. 입지 점수와 위험 요인을 확인한다.
8. 리포트의 추천 전략을 보고 일반 카페, 테이크아웃, 디저트 특화 중 방향을 판단한다.
```

성공 기준:

```text
사용자가 "이 위치는 경쟁이 강하지만 오후 체류 수요가 있어 차별화 전략이 필요하다"처럼 판단 근거를 말할 수 있다.
```

### 2.4 기존 소상공인 시나리오

```text
사용자: 이미 가게를 운영 중인 소상공인
목표: 주변 경쟁 변화와 시간대별 운영 전략 확인
```

흐름:

```text
1. 현재 가게 위치를 기준으로 상권을 선택한다.
2. 동일 업종 경쟁점 수를 확인한다.
3. 최근 신규 진입 또는 폐업 흐름을 확인한다.
4. 시간대별 유동 특성을 보고 프로모션 시간대를 정한다.
5. 3D 기반 탐색에서 가게 앞 혼잡도와 체류 포인트를 보조적으로 확인한다.
```

성공 기준:

```text
사용자가 "15시 유동이 강하니 오후 메뉴/프로모션을 강화하자"처럼 운영 액션을 정할 수 있다.
```

## 3. 핵심 기능

핵심 기능은 상권 분석이다. v0.1 개발에서는 아래 기능을 먼저 구현한다.

### 3.1 상권 지도 대시보드

목적:

```text
선택한 상권의 점포 분포와 분석 기준을 한 화면에서 제공한다.
```

기능:

```text
상권 선택
업종 필터
반경 선택: 100m / 300m / 500m
점포 마커 표시
동일 업종 경쟁점 표시
상권 요약 카드 표시
```

v0.1 완료 기준:

```text
특정 상권 1곳의 mock 또는 실데이터 점포가 지도/목록에 표시된다.
업종과 반경을 바꾸면 요약 수치가 바뀐다.
```

### 3.2 동일 업종 경쟁 강도 분석

목적:

```text
선택 위치 주변에 같은 업종이 얼마나 밀집했는지 보여준다.
```

기능:

```text
반경 내 전체 점포 수 계산
반경 내 동일 업종 점포 수 계산
업종별 분포 계산
경쟁 강도 등급 표시: 낮음 / 보통 / 높음
```

v0.1 완료 기준:

```text
카페 또는 음식점 1개 업종에 대해 반경별 경쟁 점포 수와 경쟁 강도 등급을 표시한다.
```

### 3.3 개업/폐업 흐름 분석

목적:

```text
해당 상권이 성장 중인지, 변동성이 큰지, 위험 신호가 있는지 판단한다.
```

기능:

```text
최근 기간 개업 수 표시
최근 기간 폐업 수 표시
순증감 표시
영업상태 요약
위험 요인 문장 생성
```

v0.1 완료 기준:

```text
실데이터 연동 전에는 fixture 데이터로 개업/폐업 추세를 표시한다.
실데이터 연동 후에는 인허가정보 기반으로 대체한다.
```

### 3.4 시간대별 수요/유동 특성 분석

목적:

```text
상권이 오전형, 점심형, 오후 체류형, 저녁형 중 어디에 가까운지 보여준다.
```

기능:

```text
10시 / 13시 / 15시 / 18시 관찰값 표시
사람 수 평균 또는 등급 표시
혼잡도 등급 표시
시간대별 해석 문장 표시
```

v0.1 완료 기준:

```text
수동 관찰값 또는 fixture 데이터로 시간대별 그래프와 해석 문장을 표시한다.
```

### 3.5 입지 점수

목적:

```text
여러 분석 결과를 사용자가 빠르게 비교할 수 있는 점수와 근거로 요약한다.
```

점수 항목:

```text
수요 점수
경쟁 점수
변화 점수
공간 점수
시간대 점수
총점
```

v0.1 완료 기준:

```text
규칙 기반 계산식으로 총점과 항목별 점수를 표시한다.
각 점수의 근거 문장을 함께 표시한다.
```

### 3.6 상권 해석 리포트

목적:

```text
숫자와 그래프를 창업/운영 판단 문장으로 바꿔준다.
```

기능:

```text
상권 요약
장점
위험 요인
추천 전략
다음 확인 필요 데이터
```

v0.1 완료 기준:

```text
LLM 연동 전에는 template 기반 리포트를 생성한다.
LLM 연동 후에도 점수 근거와 데이터 출처를 함께 표시한다.
```

## 4. 서브 기능

서브 기능은 핵심 상권 분석을 보조한다. v0.1에서는 핵심 기능이 먼저 동작한 뒤 연결한다.

### 4.1 Gaussian Splatting 현장 상세보기

목적:

```text
상권 지도에서 선택한 위치를 사람이 실제로 서 있는 눈높이의 3DGS 장면에서 확인한다.
```

범위:

```text
한 가게 앞 또는 한 거리 10~20m
Gaussian Splatting 현장 1개
가게/관찰 지점 마커
시간대별 혼잡도 control
통계 기반의 추상적 사람 오브젝트
```

제외:

```text
도시 전체 3D 복원
실시간 영상 스트리밍
AR
여러 상권 동시 3D 복원
```

### 4.2 사람 영역 익명화 전처리

목적:

```text
Gaussian Splatting 장면 생성을 위한 입력 이미지에서 사람 영역과 개인정보 노출 위험을 줄인다.
```

방식:

```text
경량 person detector로 bounding box 탐지
별도 전처리 프로그램에서 blur / mask / exclude 처리
정제 이미지로 3D map 생성
```

정의:

```text
이 기능은 MoE가 아니다.
lightweight person detection + preprocessing anonymization pipeline으로 정의한다.
```

### 4.3 시간대별 관찰 입력

목적:

```text
10시 / 13시 / 15시 / 18시 대표 시간대의 현장 관찰값을 상권 분석에 연결한다.
```

입력:

```text
person_count_avg
congestion_level
stay_score
movement_note
vehicle_delivery_note
observer_note
```

v0.1에서는 수동 입력을 허용하고, 자동 객체 탐지는 후속 버전으로 둔다.
